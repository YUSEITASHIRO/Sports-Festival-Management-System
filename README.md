# 🏃 寮運動会運営支援システム (Dormitory Sports Festival Management System)

### *「手計算40分を、自動集計5分へ。運営を技術でスマートに。」*
> 学生寮の運動会運営における業務効率化を実現し、参加者を待たせないスムーズな進行をサポートするツール群です。

## 📖 制作背景 (Background)

学生寮の運動会運営において、得点集計やチーム分けの「手作業による負担」が長年の課題でした。

毎年、手計算による集計には **40分以上** かかっており、その間、参加者は会場で待ちぼうけとなり、イベントの熱量が冷めてしまうという状況がありました。私はこの課題を解決するため、専門知識がなくても直感的に操作でき、現場ですぐに結果が出せるシステムを開発しました。

結果として、集計時間を **5分以内** に短縮することに成功し、運営の負担軽減と参加者の満足度向上を同時に実現しました。

## 🎯 設計思想 (Design Philosophy)

### 1. 「現場で誰でも使える」ノーコード・ノーセットアップ
学生寮の運営は毎年交代するため、複雑な環境構築が必要なシステムは定着しません。
本システムは**HTML/JSファイルのみで構成**されており、ブラウザで開くだけで動作します。サーバーやデータベースの構築は不要で、後輩への引き継ぎやすさを最優先に設計しました。

### 2. 公平性を担保するアルゴリズム
チーム分けでは、単にランダムにわけるのではなく、学年・性別比・特定イベントへの参加可否などの属性を考慮し、戦力の不均衡が起きないよう「優先度付き均等割り当て」を実装しました。

### 3. Google フォームとのシームレスな連携
当日の集計は、Google フォームからダウンロードしたCSVを読み込むだけで完結します。手入力による打ち間違いや計算ミスを排除し、信頼性の高い運営を可能にしました。

## 🔧 工夫した点 (Implementation Notes)

### チーム分けの優先順位制御
単なるランダムではなく、以下の優先順位でグループを作成するロジックを組んでいます。
1. **人数均等化**: グループ間の人数差を最小に。
2. **役割固定**: リーダーを各班に1名ずつ確実に配置。
3. **属性分散**: 学年比、性別比、特定種目参加者の偏りを自動調整。

### 不正投票の自動検知
当日、Google フォームで投票を行う際、重複投票や記名ミスが発生することがあります。
`4_2_投票データチェック.html` では、読み込んだデータを瞬時にスキャンし、不整合がある行をハイライトします。これにより、クリーンな集計結果をリアルタイムで提供可能にしました。

### 公平な集金計算
「運動会のみ参加」「後夜祭のみ参加」「記念Tシャツ購入」など、複雑な個別の参加形態を一覧管理し、支援費や共通経費を考慮した「一人あたりの請求額」を1円単位で自動算出します。

## 🚀 使い方 (Getting Started)

1. `01_名簿管理/1_グループリスト作成.html` をブラウザで開きます。
2. `sample_data/名簿データ_sample.txt` を読み込み、動作を確認します。
3. 作成した名簿を `2_チーム分け作成.html` に読み込ませ、グループを生成します。
4. 当日の集計は、`02_採点システム/` の各ファイルを手順通りに進めることで実行できます。

## 📝 制作メモ (Retrospective)

技術は「誰かの時間を生み出すため」にある。

当初は「自分が集計するならExcelでいい」と考えていましたが、後輩が引き継ぐことを考え、UI/UXをWebベースで構築することに決めました。操作の不確実性を減らすため、あえて1ファイルごとに役割を分けたステップ形式を採用しました。

実際に導入された際、集計結果をモニターに映した瞬間の「おおっ！」という歓声は、技術で課題を解決する喜びを実感させてくれました。現在もこのシステムは寮の伝統行事を支える基盤として、後輩たちに使用され続けています。
