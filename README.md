# 🏃 寮運動会運営支援システム (Sports-Festival-Management-System)

### *「手計算40分を、自動集計5分へ。運営を技術でスマートに。」*
> 学生寮の運動会運営における業務効率化を実現し、参加者を待たせないスムーズな進行をサポートするツール群です。

## 📖 制作背景 (Background)

学生寮の運動会運営において、得点集計やチーム分けの「手入力・手計算による負担」が長年の課題でした。

毎年、手計算による集計には **40分以上** かかっており、その間、参加者は会場で待ちぼうけとなり、イベントの熱量が冷めてしまうという状況がありました。私はこの課題を解決するため、専門知識がなくても直感的に操作でき、現場ですぐに結果が出せるシステムを開発しました。

結果として、集計時間を **5分以内** に短縮することに成功し、運営の負担軽減と参加者の満足度向上を同時に実現しました。

## 🎯 設計思想 (Design Philosophy)

### 1. 「現場で誰でも使える」ノーコード・ノーセットアップ
学生寮の運営は毎年交代するため、複雑な環境構築が必要なシステムは定着しません。
本システムは**HTML/JSファイルのみで構成**されており、ブラウザで開くだけで動作します。サーバーやデータベースの構築は不要で、後輩への引き継ぎやすさを最優先に設計しました。

### 2. 公平性を担保するアルゴリズム
チーム分けでは、単にランダムにわけるのではなく、学年・性別比・特定イベントへの参加可否などの属性を考慮し、戦力の不均衡が起きないよう「優先度付き均等割り当て」を実装しました。

### 3. Google フォームとのシームレスな連携
当日の集計は、Google フォームからダウンロードしたCSVを読み込むだけで完結します。手入力による打ち間違いや計算ミスを排除し、信頼性の高い運営を可能にしました。

## 🚀 各ツールの詳細と設計思想 (Detailed Tool Insights)

### 01. 名簿管理・チーム分け

#### **1_グループリスト作成.html**
- **機能**: プレーンなテキスト名簿を読み込み、UI上で「性別」「学年」「参加種目」などの属性を付与して構造化されたデータを出力します。
- **思想**: **「データの入口を整える」**。現場の生データ（ただの名前の列）を、システムが扱える「情報」へと昇華させるための基盤です。

#### **2_チーム分け作成.html**
- **機能**: 重み付き均等化アルゴリズムにより、属性の偏りを抑えた班編成を自動生成します。「リーダーを分散させる」「特定のペアを同じ班にする」といった個別調整も可能です。
- **思想**: **「納得感のある公平性」**。運営者の主観を排除しつつ、現場特有の人間関係や配慮事項をアルゴリズムに組み込むことで、全員が納得できるスタートラインを作ります。

---

### 02. 当日運営・採点システム

#### **3_イベント費用計算.html**
- **機能**: 「運動会のみ」「後夜祭のみ」「記念Tシャツ購入」など、複雑に分岐する個別の請求額を、支援費や共通経費を差し引いて1円単位で自動計算します。
- **思想**: **「祭りのあとの清算をスマートに」**。不透明になりがちなイベント会計を可視化し、金銭トラブルを未然に防ぐことで、運営の心理的ハードルを下げます。

#### **4_1_運動会結果.html**
- **機能**: 競技ごとの結果を入力すると、全チームの得点が即座に更新され、現在の暫定順位を動的に表示します。
- **思想**: **「エンターテインメントとしてのデータ」**。数字が変わるたびに一喜一憂する演出により、単なる数字の列を会場全体の熱狂へと変換します。

#### **4_2_投票データチェック.html**
- **機能**: Google フォームから出力された投票データをスキャンし、記名ミスや重複投票を瞬時に特定・ハイライトします。
- **思想**: **「公正さの守護者」**。興奮状態にある参加者のヒューマンエラーをシステム側で担保し、誰もが信頼できる集計結果を保証します。

#### **4_3_採点結果.html**
- **機能**: 全競技・全部門の最終的な集計結果を美しいUIで集約表示します。
- **思想**: **「最高のフィナーレ」**。複雑な計算過程をすべて隠蔽し、最も価値のある「結果」という数字にスポットライトを当てることで、イベントを綺麗に締めくくります。

## 🔧 実装の工夫 (Implementation Notes)

### チーム分けの優先順位制御
単なるランダムではなく、以下の優先順位でグループを作成するロジックを組んでいます。
1. **人数均等化**: グループ間の人数差を最小に。
2. **役割固定**: リーダーを各班に1名ずつ確実に配置。
3. **属性分散**: 学年比、性別比、特定種目参加者の偏りを自動調整。

## 📝 制作メモ (Retrospective)

技術は「誰かの時間を生み出すため」にある。

当初は「自分が集計するならExcelでいい」と考えていましたが、後輩が引き継ぐことを考え、操作の不確実性を減らすためにUI/UXをWebベースで構築することに決めました。あえて1ファイルごとに役割を分けたステップ形式を採用したことで、初めて触れる人でも迷わず運営できる仕組みになりました。

実際に導入された際、集計時間を大幅に短縮し、参加者が競技に集中できる環境を作れたことは、エンジニアとして「社会（コミュニティ）の課題を技術で解決する」原体験となりました。
